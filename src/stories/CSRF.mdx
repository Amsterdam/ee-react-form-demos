# CSRF

> **Warning**
> Frontend validation is only for UX, not security. It must be emphasised that much of the security enforcements needs to happen on the backend, however, frontend forms can help guide safe input and prevent common vulnerabilities.

A malicious site tricks a logged-in user into submitting a request to your site without their knowledge (e.g. clicking a link that triggers a form `POST` request).

## Flow

- A CSRF token is generated by the backend and sent to the frontend (often in a cookie or a hidden input).
- When the form is submitted, the token is sent to the backend (with the form data or as a request header).
- The backend verifies that the token matches the user session and originated from your domain. Therefore, if there is no token present or it has an invalid value, the backend should reject the request.
