# UX Patterns

| Pattern                                 | What it is                                                     | Why it helps                                       | Example                                                  | Tips                                                                        |
| --------------------------------------- | -------------------------------------------------------------- | -------------------------------------------------- | -------------------------------------------------------- | --------------------------------------------------------------------------- |
| **Multi-Step Forms (Wizards)**          | Break a long form into smaller, manageable steps.              | Reduces cognitive load, improves completion rates. | Checkout → Shipping → Payment → Confirmation.            | Show progress indicator, allow back navigation, persist data between steps. |
| **Unsaved Changes Prompts**             | Warn user before leaving if changes aren’t saved.              | Prevents accidental data loss.                     | Profile edit → “Are you sure you want to leave?” dialog. | Trigger only if state changed, use native or custom modal.                  |
| **Form Autosuggestions / Autocomplete** | Suggest values while typing.                                   | Saves time, reduces typos.                         | Address autocomplete via API.                            | Debounce API calls, highlight matched text.                                 |
| **Inline Validation**                   | Show validation feedback as user types.                        | Prevents frustration, immediate error correction.  | Email field showing “Invalid email” before submit.       | Validate after blur for complex inputs, use friendly tone.                  |
| **Progressive Disclosure**              | Show additional fields only when relevant.                     | Keeps UI clean and focused.                        | “Company name” only if “Business account” is selected.   | Animate reveal, maintain accessibility.                                     |
| **Smart Defaults & Prefilled Values**   | Provide sensible default values or prefill from profile.       | Reduces user effort.                               | Country defaults to user’s location.                     | Allow easy editing, don’t over-assume.                                      |
| **Field Grouping & Logical Sections**   | Organize related inputs under headings.                        | Improves readability, reduces overwhelm.           | “Personal Info”, “Payment”.                              | Limit to \~4–6 fields per group, use whitespace.                            |
| **Save & Resume Later**                 | Let users save partially completed forms.                      | Improves completion for long forms.                | Job application form drafts.                             | Store drafts in backend/local storage, send reminders.                      |
| **Inline Help & Tooltips**              | Provide short guidance near tricky fields.                     | Reduces confusion.                                 | “Password must be 12+ chars and include a symbol.”       | Keep concise, use tooltips for optional info.                               |
| **Skeletons / Loading States**          | Show placeholders while fetching initial data.                 | Improves perceived performance.                    | Grey boxes before form loads.                            | Disable fields until ready, show loading indicator.                         |
| **Confirmation Screens**                | Let users review all data before final submission.             | Prevents mistakes, builds trust.                   | Review order before paying.                              | Include “Edit” buttons, highlight key info.                                 |
| **Accessible Error Handling**           | Make error messages screen-reader friendly and visually clear. | Ensures inclusivity, meets WCAG.                   | Errors linked with `aria-describedby`.                   | Use color *and* text, place errors close to fields.                         |
