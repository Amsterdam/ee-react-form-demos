## Zod/JS validation vs Browser validation - Pros + Cons

### Browser validation

- Con: Browser Validation only handles individual errors - slower + more tedious
- Con: Required errors displayed on load before user has touched inputs
- Con: Increases complexity in code
  - Error handling on submit on on change
  - Testing
  - Schema/validation handling
  - Error message handling

### JS validation

- Con: Becomes complex for larger forms quickly, especially for child fields with repeater/variable types of data
- Con: Can conflict with browser validation so you should use noValidate + avoid required attribute, which disagrees with browser accesibility... See [GitHub issue](https://github.com/whatwg/html/issues/7848)
