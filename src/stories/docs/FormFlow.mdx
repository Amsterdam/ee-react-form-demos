# Form Submission Flow

This document explains what happens when you submit a form in a browser.

1. [Traditional HTML forms](#traditional-html-forms)
2. [React SPA forms](#react-spa-forms)
3. [React Server Components + Server Actions](#forms-with-react-server-components--server-actions)

## Traditional HTML forms

Traditionally in the browser, when a user clicks **Submit** in a form, the browser performs a full HTTP form submission and navigates away from the current page.

1. The HTML `form` element specifies:
  - the target URL (`action` attribute)
  - the HTTP method (`method` - usually `GET` or `POST`)

  ```html
  <form action="/submit" method="POST">
    <input name="username" value="john">
    <input name="email" value="john@example.com">
    <button type="submit">Submit</button>
  </form>
  ```

2. The user clicks submit (or presses Enter) - which triggers an event that the browser intercepts.
3. The browser collects all the form field values and encodes them based on the form's `enctype`. The `enctype` value can be:
    - `application/x-www-form-urlencoded` (default): `username=john&email=john%40example.com`
    - `multipart/form-data` (for file uploads): Creates `FormData` with boundaries
    - `text/plain` (rarely used): Plain text format
4. The browser constructs and sends a HTTP request:
    - `GET` - Values are appended to the URL query string
    - `POST` - Values are sent in the requst body
    - Sets appropriate `Content-Type` header based on the encoding (#3)
5. The browser navigates away from the current page and waits for a server response (the backend)
6. The server (backend) processes the request and can respond with:
    - A new HTML page - Browser renders the new page
    - A redirect (3xx status) - Browser follows the redirect
    - An error page - Browser displays the error(s)

## React SPA forms

In modern React single-page applications (SPAs), a full form submission is usually avoided because it triggers a full navigation and resets application state.

Instead, via JavaScript we can intercept the form submission and send the data asynchronously and then update the UI without any navigation.

### Example

Configure the HTML `form` element and attach a submit handler.
  - The `onSubmit={handleSubmit}` tells form submission events to execute the `handleSubmit` function
  - In `handleSubmit` we call `e.preventDefault();` to stop the browser's default form submission behaviour (and page reload).
  ```tsx
    const handleSubmit = (e: FormEvent<HTMLFormElement>) => {
      e.preventDefault();

      const formData = new FormData(e.currentTarget);
      const validationErrors = validateForm(formData);
      setErrors(validationErrors);

      ...
    };

    return (
      <Grid paddingBottom="x-large" paddingTop="large">
        <Grid.Cell span={{ narrow: 4, medium: 8, wide: 6 }}>
          <Heading
            level={1}
            size="level-3"
            className="ams-mb-m"
            data-testid="heading"
          >
            Form title
          </Heading>

          {/* Use noValidate so browser validation doesn't block JS */}
          <form
            className={`${styles.form} ams-gap-m`}
            onSubmit={handleSubmit}
            data-testid="form"
          >
            {/* form fields */}
          </form>
  ```

### Flow

1. A submit handler is attached `onSubmit={handleSubmit}`
2. The user submits the form. This triggers the `onSubmit` event, which executes the custom `handleSubmit` function.
3. `e.preventDefault()` in the `handleSubmit` function stops the browser from performing its default submission behaviour.
4. The form data is collected. Data can come from:
    - The DOM via new `FormData(e.currentTarget)` (good for small/medium forms)
    - `React` state (common when fields are fully controlled components).
    - libraries like `react-hook-form`
5. Perform client-side validation (e.g. with [Zod](https://zod.dev/), [Yup](https://github.com/jquense/yup) or custom logic).
6. If the submission is valid, send the data to the backend. Usually, an API request with `fetch` or [Axios](https://axios-http.com/).
    - Use **JSON** for standard text-based payloads.
    - Use `FormData` (with `multipart/form-data`) if uploading files or sending binary data.
7. The UI is updated based on the API response:
    - Show success
    - Show error(s)

## Forms with React Server Components + Server Actions

With React Server Components (RSC) and frameworks like [Next.js](https://nextjs.org/) App Router, forms can submit directly to server-side functions called Server Actions. Read more about [React 19 - React Server Components (RSC) & Server Actions](../?path=/docs/docs-react-react-19-react-server-components-rsc-server-actions--docs).

This model restores the simplicity of native HTML forms but still enables modern UX patterns such as partial UI updates, optimistic states and loading indicators.

### Example

```ts
// app/actions.ts
"use server";

export async function submitForm(formData: FormData) {
  const username = formData.get("username");
  const email = formData.get("email");

  // Validate and persist (database, API call etc)
}
```

```tsx
// app/page.tsx (Server Component)
import { submitForm } from "./actions";

export default function Page() {
  return (
    <form action={submitForm}>
      <input name="username" />
      <input name="email" />
      <button type="submit">Submit</button>
    </form>
  );
}
```

### Flow

1. The form's `action` attribute is bound to a server action function.
2. The user submits the form.
3. The browser collects and encodes form values, similar to a traditional HTML form.
4. The request is sent to the server, and the server action executes:
  - read values from `FormData`
  - performs validations
  - persists data (database, API etc.)
5. The server returns a new render result.
6. React updates the UI.

## Comparison

### Traditional HTML forms

- Simplest architecture
- Full page navigation on submit
- Minimal JavaScript required

### React SPA forms

- Best for highly interactive UIs
- Requires JavaScript and API endpoints
- No page reload (state is preserved)

### React Server Actions

- Combines the best of both worlds
- Forms submit naturally without manual fetch()
- Server-side validation is built into the model
- Progressive enhancement is possible
