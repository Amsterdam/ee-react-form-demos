# CSRF

> **Warning**
> Frontend validation is only for UX, not security. It must be emphasised that much of the security enforcements needs to happen on the backend, however, frontend forms can help guide safe input and prevent common vulnerabilities.

This document explains **Cross-Site Request Forgery (CSRF)** and how to prevent it.

CSRF is an attack where a malicious website tricks a user’s browser into making an authenticated request to your application without the user’s knowledge.
Because the browser automatically includes cookies, the request may be processed as if it was intentionally made by the logged-in user.

For example: a user is logged into `bank.com`, then visits `evil.com`, which silently triggers a `POST` request to `bank.com/transfer-money`.

> CSRF mainly affects applications that rely on **cookie-based authentication**.

## Flow

- The backend generates a CSRF token and provides it to the frontend (commonly via a cookie, hidden input, or page load response).
- When the user submits the form, the token is sent to the backend (alongside the form data or as a request header).
- The backend verifies the token is valid for the user/session.
- If the token is missing or invalid, the backend rejects the request.

## Examples

### Unsafe - sending data to the backend without a CSRF token

```tsx
// BAD: No CSRF token included
fetch('/api/form-submission', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'John' }),
});
```

### Safe - sending data to the backed with a CSRF token

```tsx
// Assume csrfToken is provided by backend via page load or cookie
fetch('/api/form-submission', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRF-Token': csrfToken,
  },
  body: JSON.stringify({ name: 'John' }),
});

```

With plain forms:

```tsx
<form method="POST" action="/update-profile">
  <input type="hidden" name="_csrf" value={csrfToken} />
  <input name="name" />
  <button type="submit">Save</button>
</form>
```

Note that with React/SPAs a hidden input shouldn't be necessary, you can use the state to store and handle the token values.
