# React (JSX) vs HTML

This page documents common differences between **HTML attributes** and **JSX (React) props**.

It is intended as a quick reference for developers moving between plain HTML and React, where many attributes use different naming conventions or behave differently in controlled components.

| Feature / Element              | **HTML**                                        | **JSX (React)**                                                                                                                 |
| ------------------------------ | ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `for` attribute in `<label>`   | `for="username"`                                | `htmlFor="username"`                                                        |
| Input value binding            | `<input value="some text">`                     | Controlled: `value={state}` + `onChange`; Uncontrolled: `defaultValue="..."` |
| Checkbox/radio state           | `<input type="checkbox" checked>`               | Controlled: `checked={state}` + `onChange`; for initial state: `defaultChecked` |
| `<textarea>` content           | Wraps inner text: `<textarea>Hello</textarea>`  | Uses `value="..."` prop instead (value-only control via prop) |
| Numeric attributes (len, etc.) | `maxlength="10"` (string)                       | `maxLength={10}` (camelCase prop, number) or `"10"` |
| Self-closing form tags         | Can close `<input>` as `<input>` or `<input />` | Must use `<input />`, the self-closing slash is required |
| Form data handling             | Browser submits directly (page reload)          | JS handles submission, data captured via state, refs, or `FormData` — more manual control                                       |
