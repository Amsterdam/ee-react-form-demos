Most form fields use components from [amsterdam-design-system](https://designsystem.amsterdam/), however, we also have a wrapper around [react-select](https://react-select.com) - and some 'repeater' components (e.g. `AnnotationRepeaterRow`) between the [amsterdam-design-system](https://designsystem.amsterdam/) form component and our parent form.

We adopted a **bottom-up approach** to writing the tests - meaning we test the individual components first and work up to the parent `CreateEntity.tsx` component. We followed this approach:

## 1. Foundation components

Begin with the individual form components since they're the building blocks:

**Priority Order:**

1. **FormTextInput, FormTextarea, FormSelect** - These are likely your most straightforward components
2. **FormCheckboxInput, FormAutoSelect** - Slightly more complex but still isolated
3. **AnnotationRepeaterRow, LinkRepeaterRow** - Test the row components before the containers
4. **AnnotationRepeater, LinkRepeater** - Test the repeater logic after rows work

**Testing Strategy for Components:**

- Test props acceptance and rendering
- Test user interactions (typing, selecting, checking)
- Test validation states (if handled at component level)
- Test accessibility (labels, ARIA attributes)
- Mock any external dependencies

## 3. Integration Testing Strategy

For the main `CreateEntity` container, create separate test suites:

1. CreateEntity (HTML5 version) - Test basic form submission, field interactions
2. CreateEntity (HTML5 + Zod) - Test schema validation integration
3. CreateEntity (react-hook-form + Zod) - Test RHF integration

The focus for testing the container should be on:

- Form submission with valid data
- Validation error handling and display
- Field interactions and state updates
- Repeater add/remove functionality
- Error scrolling behavior
