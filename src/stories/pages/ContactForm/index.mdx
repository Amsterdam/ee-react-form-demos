## Overview

This demo shows a basic contact form with three required fields:

- **Name**
- **Email** (must be valid format)
- **Message**

## Purpose

This Contact Form 'on Submit' variant demonstrates the most basic validation strategy: all fields are validated only when the form is submitted. It’s often the simplest implementation and is commonly used in quick prototypes or small forms.

- All fields must be filled out.
- The email must be in a valid format.
- Error messages are only displayed after the user clicks Submit.

## Key Points

- **Low friction while typing**: The user can fill in the form without any distractions from early validation messages.
- **Batch feedback**: All validation errors are shown at once after the submit attempt.
- **Simple implementation**: This approach is straightforward to wire up.

- **Risk of frustration**: If multiple fields are incorrect, the user only finds out after completing the entire form.
- **Delayed feedback**: Typos or mistakes aren’t caught until the end, which can increase the number of corrections needed.
- **Best fit**: Works fine for short forms like this one, but becomes less ideal as forms grow in length or complexity.
