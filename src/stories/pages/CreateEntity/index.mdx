## Overview

This demo implements a complex “Create Entity” form that collects data for generating a [Backstage](https://backstage.io/) entity YAML file.

Fields

- **Kind** – select menu (required)
- **Name** – input (required)
- **Description** – textarea
- **Type** – select menu (required)
- **Lifecycle** – select menu (required)
- **Owner** – react-select (single value, required)
- **Has System?** – checkbox
- **System** – `react-select` (required only if `Has System` is checked)
- **Tags** – `react-select` (multiple values)
- **Annotation Repeater** – dynamic rows:
  - Key – `react-select` (required)
  - Value – input/select (required)
- **Links Repeater** – dynamic rows:
  - URL – input (required)
  - Title – input (required)
  - Icon – select (required)

## Purpose

This variant uses native HTML5 validation and unlike [Booking Form](../?path=/docs/plain-html5-bookingform-html5-validation--docs) it lets the browser handle all validation. Whilst this solution provides a clean frontend codebase, with large forms HTML5 validation shows its limitations with the UX. For example, users will only see the first error that the browser encounters - so multiple errors can require multiple submit attempts.

- Required fields (Kind, Name, Type, Lifecycle, Owner).
- `System` required if `Has System` is checked.
- Repeater rows (Annotations and Links) must be valid for all added rows.
- Each repeater item has its own required subfields.

## Key Points

- **Pro**: **Built-in simplicity**: Using required, type=email/url, and pattern attributes provides quick wins with no extra libraries.
- **Con**: **Progressive blocking**: Browser validation halts at the first invalid field, forcing the user to fix issues one at a time.
- **Con**: **Poor UX scaling**: With so many fields, this quickly becomes frustrating and confusing for users.
- **Con**: **Dynamic arrays**: Adding repeater rows makes native validation even messier, as the browser only surfaces one invalid item at a time.
- **Con**: **Dynamic repeaters** require custom scripting to ensure proper validation (HTML5 alone doesn’t handle them gracefully).
