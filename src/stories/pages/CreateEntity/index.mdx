## Overview

This demo implements a complex “Create Entity” form that collects data for generating a [Backstage](https://backstage.io/) entity YAML file.

Fields

- **Kind** – select menu (required)
- **Name** – input (required)
- **Description** – textarea
- **Type** – select menu (required)
- **Lifecycle** – select menu (required)
- **Owner** – react-select (single value, required)
- **Has System?** – checkbox
- **System** – `react-select` (required only if `Has System` is checked)
- **Tags** – `react-select` (multiple values)
- **Annotation Repeater** – dynamic rows:
  - Key – `react-select` (required)
  - Value – input/select (required)
- **Links Repeater** – dynamic rows:
  - URL – input (required)
  - Title – input (required)
  - Icon – select (required)

## Purpose

This variant uses [Zod](https://zod.dev/) schema validation instead of native HTML5 validation, giving full control over error handling, conditional logic and repeatable fields.

### Validation Rules

- Required fields (Kind, Name, Type, Lifecycle, Owner).
- `System` required if `Has System` is checked.
- Annotation rows: Key + Value required.
- Link rows: URL, Title, Icon required.
- All invalid fields are surfaced together after submit.

## Key Points

- **Full control over UX**: Validation messages are shown inline or in summary alerts, styled consistently with the design system.
- **Better user flow**: Users can see all validation issues at once, instead of being blocked by the first error (as is the case in [CreateEntity (HTML5 Validation)](../?path=/docs/plain-html5-createentity-html5-validation--docs)).
- **Schema-driven validation**: By defining validation rules in one place, the logic is easier to maintain, extend and reason about.

### Pros

- **Scalable** - Works well as forms grow in complexity, or when repeaters allow dozens of dynamic items.
- **Consistent messaging** - Errors are shown in a uniform style, independent of browser quirks.
- **Future-proof** - Schema validation can be reused across frontend + backend, ensuring consistency.

### Cons

- **Setup cost** - Requires extra code compared to just adding required attributes.
- **Performance** - Very large forms with many dynamic arrays might benefit from incremental validation instead of full schema checks every time.
