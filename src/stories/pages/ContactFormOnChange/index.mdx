## Overview

This demo shows a basic contact form with three required fields:

- **Name**
- **Email** (must be valid format)
- **Message**

## Purpose

This Cntact Form 'on Change' variant demonstrates a more user-friendly validation strategy by validating fields as the user interacts with them. It balances real-time feedback with preventing error “spam” by using two states:

- **Touched**: whether an individual input has been interacted with.
- **SubmitTouched**: whether the user has attempted to submit the form.

- All fields must be filled out.
- The email must be in a valid format.
- Inline errors appear once a field has been touched and contains invalid input.
- A global error alert is shown only after the user attempts to submit.

## Key Points

- **Immediate feedback**: Users catch mistakes as they type, instead of waiting until the end.
- **Error gating**: Errors aren’t shown prematurely — only once a field has been touched, or after a submit attempt.
- **Balanced UX**: This approach avoids overwhelming the user with red error states before they’ve had a chance to provide input.

- **Friendly experience**: Inline validation reduces frustration by pointing out errors at the moment they occur.
- **Clear error visibility**: The global error alert after submit ensures users can quickly identify all remaining issues in one place.
- **Consistency**: Using both inline and summary errors supports different user preferences and improves accessibility.
