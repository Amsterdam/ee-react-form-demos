# File Handling

> **Warning**
> Frontend validation is only for UX, not security. It must be emphasised that much of the security enforcements needs to happen on the backend, however, frontend forms can help guide safe input and prevent common vulnerabilities.

## Risks

- Uploading dangerous files (for example, `.exe`, `.php`)
- Malicious files disguised with a safe extension (`.jpg` containing script code)
- Large files (DoS attempts)

## Frontend best practices

- Restrict file types in the `<input>`:
```tsx
<input type="file" accept=".jpg,.png,.pdf" />
```
- Provide file size checks via JavaScript before uploading.
- Avoid reading file content in the browser.

## Backend must

- Validate file and MIME types server-side
- Scan for malware
- Store files outside of the public web root (to prevent [directory traversal attacks](https://en.wikipedia.org/wiki/Directory_traversal_attack))
