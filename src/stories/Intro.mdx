# Gemeente Amsterdam Form Demos

> **Warning**
> This project is a work in progress so will likely change in the future.

Forms are one of the most common ways users interact with our applications - and one of the complex parts of frontend development to get right. From accessibility and validation to design consistency and user experience, there are many parts to consider.

This project combines best practices, demos and example implementations based on Gemeente Amsterdam projects and internal discussions. It's intended to help developers and designers choose the right aproach for their needs and build functional forms that are also accessible and maintainable.

Frontend applications in Gemeente Amsterdam use the component library [React](https://react.dev/). We've standardised two form options - pick the one that best fits your project:

- **HTML5** for quick, simple forms
- **React Hook Form** when you need advanced validation, performance and flexibility.

For each demo we also try to include documentation with the following structure:

- General description
- Background/useful to know caveats
- Overview of the files required
- And the code from each file

    - Form state vs UI state (what goes in a form, what belongs in app state)

      - File handling
      - CSRF
